# 末世异闻录写作协作

围绕《末世异闻录》进行续写、修文、设定补完、章节重构时，先按这里的流程工作，再动正文或设定文件。

这份 skill 已经吸收了一部分长篇连载 workflow 的优点，但只保留对本项目真正有用的部分：

- 先做章节执行包，再开写
- 写完必须过审查闸门
- 需要能查询设定、人物、伏笔和上下文
- 中断以后要能顺着恢复，而不是每次重读全部资料
- 流程服务《末世异闻录》，不照搬别人的网文模板

## 快速流程

1. 先确认任务类型。
   - 续写正文
   - 重构章节
   - 补设定
   - 统一人物口癖与对话
   - 调整卷结构或剧情钩子
2. 先读总纲和作者备忘，再按任务补读对应资料。
3. 写之前先列出本次不能撞的红线。
4. 落笔时优先写场面、行动、秩序、代价、关系变化，不要先写抽象判断。
5. 写完后反查句式、模板感、人物功能重复、世界规则是否落地。

如果任务更复杂，直接切到下面五种工作模式之一：

- `规划模式`
- `续写模式`
- `审查模式`
- `查询模式`
- `续接模式`

具体见 `references/workflow-modules.md`

## 工作原则

- 保持中文写作。
- 保持“黑暗背景下积极推进”的总气质。
- 主角的现代人视角主要通过吐槽、判断、办法、改造欲和落地措施显影，少写悬空独白。
- 设定服务正文，正文服务人物和推进，不要让设定停在讲解层。
- 到一个新地方时，除了新怪事，还要有新秩序、新利益、新偏见、新代价或新选择。
- 新日教前期要有真实救人能力和秩序魅力，问题要藏在扩张、解释权、登记权、收容权里，慢慢揭。
- 赫莱娅不能长期停在点火和神选功能位，要持续落到成长、选择、冒险心和责任感。
- 林砚不能只当嘴替。他要持续显出异乡感、现代视角、判断力和未来统治者的雏形。
- 章节推进不能只靠“想到哪写到哪”。复杂章节要先做执行包，再落正文。
- 审查不是挑语病，而是先查人物、设定、承接、兑现、钩子和节奏。
- 断点恢复时优先看上次留下的任务状态、章节目标和未回收项，不要每次从零读完整工程。

## 写作时强制避免

- 不要高频使用 `谁怎样怎样` 的提纲句。
- 不要高频使用 `哪怎样怎样` 的悬空概括。
- 不要高频使用 `不是……而是……`
- 不要高频使用 `先……再……`
- 不要高频使用 `像……`
- 不要高频使用“为什么……”式标题和说明句。
- 不要把设定稿写成问答腔、拆书腔、AI分析腔。
- 不要把段落写成固定模板：到新节点，见异常，见秩序冲突，林砚识别问题，众人协作止血，最后抬高主题。
- 不要为了厚重感一味压阴郁。

具体替代方式见 `references/language-guardrails.md`。

## 工作模式

### 规划模式

用于卷规划、章节重构、20 章以上剧情编排。

核心要求：

- 先明确这一段的阶段任务
- 拆清人物线、秩序线、真相线分别推进什么
- 保证每一段不是只换地图，还要换关系、代价或认知

### 续写模式

用于直接写章节正文。

核心要求：

- 开写前先做章节执行包
- 只带本章必需的设定，不把资料全灌进正文
- 写完必须过一轮审查闸门

### 审查模式

用于评估一章或一段文字能不能接进当前工程。

核心要求：

- 先查硬冲突，再查人物，再查节奏
- 审查重点是功能和后果，不是修辞洁癖
- 需要指出哪一段该删、该换、该前移、该后移

### 查询模式

用于快速回答“这个人物之前到哪了”“这个设定在哪份文件里”“这条伏笔收没收”。

核心要求：

- 先锁定问题类型
- 只读必要文件
- 输出要短，优先告诉当前能直接拿来用的结论

### 续接模式

用于接上次写到一半的活，或者把这次工作交给下一次协作。

核心要求：

- 记录上次做到哪一步
- 记录当前章目标、已兑现项、未回收项、需要补读的文件
- 下次续上时，优先恢复状态，不再大面积重复阅读

## 续写正文时的落地顺序

1. 先判断这一章真正推进的是人物、秩序、真相，还是三者一起推进。
2. 把世界设定压进具体事情里。
   - 税
   - 配给
   - 路权
   - 城防
   - 学派规矩
   - 教会接口
   - 虫群治理术语
3. 让人物在行动和对话里显出差异，不要靠作者总结人物性格。
4. 章末优先留下人物关系变化、代价、决断或真相回声，不要只留“下一站去哪”。

复杂章节在开写前，先按 `references/chapter-contract.md` 生成一份最小执行包。

## 补设定时的落地顺序

1. 先写结构、条件、作用、代价、风险、限制。
2. 再写这项设定会怎样进入正文。
3. 最后补和现有总纲、人物线、宗教线、虫群线、新旧冲突线的接缝。

如果一项设定解释完以后仍然很像作者在回答读者提问，就继续改，直到它更像世界资料而不是答疑。

## 审查闸门

正文或重构段落写完后，至少过下面五道：

1. 人物有没有继续往前长
2. 设定有没有和现有资料打架
3. 这一章有没有兑现，不只是加谜
4. 结尾留下的是关系、代价、决断还是只有“下一站”
5. 句式和模板感有没有明显露头

详细检查项见 `references/chapter-contract.md`

## 断点续接

如果工作做到一半被打断，结束前至少记下：

- 当前任务是什么
- 做到哪一步
- 下一步最应该接什么
- 哪些文件必须先重读
- 哪些坑已经挖了但还没收

续接格式见 `references/session-resume.md`

## 完稿自检

- 这一段有没有落到具体人、具体事、具体秩序。
- 这一章有没有真正推进人物。
- 世界规则有没有踩到人物身上。
- 有没有重复使用熟悉句模。
- 赫莱娅和林砚有没有继续长，不只是继续会说话。
- 设定有没有变成背景板，或者反过来压过剧情。
- 读者记住的会不会只是信息，而不是场面和关系。
